About the vulnerabilities: Sensitive Data Exposure

Massive data breaches in the past years as well as governments regulations such as GDPR make handling of user data a critical aspect of an application's operation.

Over the last few years, this has been the most common impactful attack. The most common flaw is simply not encrypting sensitive data. When crypto is employed, weak key generation and management, and weak algorithm, protocol and cipher usage is common, particularly for weak password hashing storage techniques. For data in transit, server side weaknesses are mainly easy to detect, but hard for data at rest..

Challenge Tips and Tricks

Along with the 'secret' file you will find many other files in the bucket including the html for a chat application that the hackers use. The users file for the application can be found in the same bucket and the passwords are stored as unsalted SHA-1 hashes.
The users of the system have strong passwords but the test account not so much. Crack the test account password and login into the chat to pass this challenge.
